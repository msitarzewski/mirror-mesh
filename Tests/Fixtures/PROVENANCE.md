## What this fixture proves — and what it does **not**

This clip exists for **one purpose only**: to exercise the `FileFrameSource` →
`AVAssetReader` → BGRA `CVPixelBuffer` → `Pipeline` code path in headless CI without
requiring a camera.

It **does not** prove the live Vision landmark pipeline finds real faces, because the
fixture has **no human likeness** — Apple Vision will (correctly) not detect a face in
two dots and an ellipse. The original M15 spec called for a real-face clip; we instead
chose the synthetic path because we will not commit footage of any real person without
explicit, documented consent, and we will not commit third-party footage without an
audit trail of its license.

Two consequences:

1. The original M15 acceptance criterion *">50% of frames produce non-nil landmarks"*
   is **explicitly removed**. The fixture test asserts only that frames flow through
   `FileFrameSource` with correct dimensions and monotonic frame IDs. That's the
   actual contract of `FileFrameSource`.
2. Real-face validation of `LandmarkExtractor` remains on M13's manual-test checklist
   (`memory-bank/release/v0.2.0/M13-live-camera.md`), to be exercised against the
   developer's own webcam at runtime — never committed.

## How to add a different fixture

A future fixture clip from a real face is acceptable **only** if:

1. It depicts the committer (selfAsSource consent — the only consent scheme we
   currently support, per `ConsentScheme.selfAsSource`)
2. A `ConsentRecord` JSON sidecar is committed alongside it with `accepted_at`,
   `user_disclosure_text_sha256`, and a signed assertion
3. License is MIT or CC-BY-4.0 with attribution recorded here
4. File size stays ≤ 8 MB per M15 spec

Until that workflow exists, the procedural fixture is the only acceptable artifact.
