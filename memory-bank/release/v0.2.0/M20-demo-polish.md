## Objective

The shippable face of v0.2.0: README that makes the project comprehensible in 30 seconds, screenshots that show the live pipeline running with watermark + landmarks + avatar, and a one-command "paper figures" export that generates the latency-vs-stage plot the paper needs.

## Deliverables

- `README.md` rewritten with:
  - Hero screenshot (the `.app` window running live)
  - 30-second elevator pitch
  - Quickstart (build, test, run)
  - Pointer to `docs/` and `memory-bank/`
  - Status badge
  - License (resolved per ADR-0005 — set it for v0.2.0 or document the open decision)
- `docs/screenshots/` — pipeline running, settings panel, telemetry panel, recorded `.mov` in QuickTime with badge visible
- `docs/figures/` — paper-ready PDFs/PNGs generated by `bench/scripts/figures.py` from JSONL traces:
  - `latency_by_stage.pdf` — P50/P95/P99 per stage, bar chart
  - `e2e_distribution.pdf` — end-to-end histogram
  - `power_by_session.pdf` — power per session
- `bench/scripts/figures.py` — `matplotlib`-based figure generator (matplotlib install via `pip install matplotlib`, documented in `docs/figures.md`)
- `CHANGELOG.md` populated with v0.1.0 + v0.2.0 entries

## Verification

```bash
# Quickstart from a fresh shell on the reference machine
git clone <repo> mirror-mesh && cd mirror-mesh
export DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer
swift test
swift run mirrormesh-bench --scenario bench/scenarios/fixture.json
python3 bench/scripts/figures.py bench/out/fixture_*.jsonl
open docs/figures/latency_by_stage.pdf
```
